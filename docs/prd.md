# 초등학생 구구단 연습장 PRD
버전: v0.1  
문서 목적: 초등학생이 쉽고 재미있게 구구단을 반복 학습할 수 있는 웹앱의 초기 제품 요구사항 정의  
대상 독자: 기획자, 디자이너, 프론트엔드 개발자, AI 코딩 에이전트

---

## 1. 제품 개요

### 1.1 제품명
구구단 연습장

### 1.2 한 줄 설명
초등학생이 밝고 예쁜 화면에서 구구단을 반복 연습하고, 틀린 문제를 다시 풀 수 있는 웹앱

### 1.3 제품 목표
이 제품의 목표는 구구단 학습을 “지루한 암기”가 아니라 “쉽고 반복 가능한 놀이형 학습 경험”으로 바꾸는 것이다.

핵심 목표는 아래와 같다.

1. 아이가 혼자서도 쉽게 시작할 수 있어야 한다.
2. 문제를 보고 바로 답을 고를 수 있어야 한다.
3. 정답과 오답을 즉시 확인할 수 있어야 한다.
4. 틀린 문제를 다시 풀 수 있어야 한다.
5. 전체 경험이 밝고 예쁘고 부담 없어야 한다.

---

## 2. 배경과 문제 정의

구구단은 반복 학습이 핵심이지만 종이 학습지나 단순 암기 방식은 아이들이 쉽게 지루함을 느낀다.  
또한 보호자가 계속 옆에서 채점하거나 진도를 봐주지 않으면 반복 학습이 끊기기 쉽다.

현재 문제는 아래와 같다.

### 2.1 반복 학습이 지루하다
구구단은 많이 풀어봐야 익숙해지는데, 일반적인 방식은 재미 요소가 부족하다.

### 2.2 즉시 피드백이 부족하다
아이 입장에서는 맞았는지 틀렸는지 바로 확인하는 경험이 중요하다.

### 2.3 틀린 문제 복습이 번거롭다
틀린 문제를 따로 모아 다시 풀게 만드는 구조가 없으면 실수가 반복된다.

### 2.4 혼자 사용하기 어렵다
입력 방식이 어렵거나 화면이 복잡하면 보호자 도움 없이 사용하기 힘들다.

이 프로젝트는 위 문제를 해결하기 위해, 아주 단순한 흐름과 즉시 피드백 중심의 구구단 연습 경험을 제공한다.

- 아이가 혼자서도 사용할 수 있는 간단한 학습 웹앱을 원하는 부모

### 3.3 사용자 특성
- 긴 설명보다 짧고 직관적인 UI가 필요함
- 키보드 입력보다 버튼 선택이 쉬움
- 칭찬과 시각적 보상이 동기 유지에 도움 됨
- 복잡한 메뉴나 옵션은 오히려 방해됨

## 8. 핵심 기능 요구사항

## 8.1 시작 화면

### 목적
아이가 연습할 단을 고르고 쉽게 시작할 수 있어야 한다.

### 요구사항
- 서비스 제목이 보여야 한다.
- 짧은 안내 문구가 있어야 한다.
- 2단~9단, 전체 섞기 버튼이 있어야 한다.
- 단 선택 전에는 시작 버튼이 비활성화되어야 한다.
- 단 선택 후 시작 버튼이 활성화되어야 한다.

### 완료 기준
- 단 선택이 명확하게 반영된다.
- 시작 버튼 클릭으로 문제 화면으로 이동한다.

---

## 8.2 단 선택 기능

### 목적
원하는 단만 집중적으로 연습하거나 전체 단을 섞어서 연습할 수 있어야 한다.

### 요구사항
- 아래 선택지를 제공한다.
  - 2단
  - 3단
  - 4단
  - 5단
  - 6단
  - 7단
  - 8단
  - 9단
  - 전체 섞기
- 선택된 단은 시각적으로 구분되어야 한다.
- 한 번에 하나만 선택 가능해야 한다.

### 완료 기준
- 단 선택 상태가 명확하게 유지된다.
- 시작 시 해당 조건으로 문제 배열이 생성된다.

---

## 8.3 문제 출제 기능

### 목적
선택한 단에 맞는 구구단 문제를 랜덤으로 제공해야 한다.

### 요구사항
- 기본 문제 수는 10문제로 한다.
- 특정 단 선택 시 해당 단만 출제한다.
- 전체 섞기 선택 시 2단~9단 범위에서 랜덤 출제한다.
- 문제는 `A × B = ?` 형태로 표시한다.
- 보기 4개 중 하나는 정답이어야 한다.
- 보기 순서는 랜덤이어야 한다.

### 완료 기준
- 문제 10개가 정상 생성된다.
- 현재 문제를 화면에 정확히 표시한다.
- 보기 4개가 정상 노출된다.

---

## 8.4 답안 선택 및 채점

### 목적
아이가 쉽게 답을 선택하고 즉시 결과를 확인할 수 있어야 한다.

### 요구사항
- 답은 객관식 4지선다 버튼으로 입력한다.
- 사용자가 보기를 누르면 즉시 채점한다.
- 정답이면 긍정 피드백을 보여준다.
- 오답이면 정답을 함께 안내한다.
- 한 문제는 한 번만 채점되어야 한다.
- 채점 후에는 보기 버튼이 비활성화되어야 한다.

### 완료 기준
- 정답/오답 판정이 정확하다.
- 중복 채점이 발생하지 않는다.
- 채점 후 다음 문제 버튼이 활성화된다.

---

## 8.5 진행률 및 점수 표시

### 목적
현재 얼마나 진행했는지 아이가 쉽게 이해할 수 있어야 한다.

### 요구사항
- 현재 몇 문제째인지 보여준다.
- 총 문제 수를 보여준다.
- 맞힌 개수와 틀린 개수를 보여준다.
- 진행률을 시각적으로 표현할 수 있다.
- 정답률은 결과 화면에서 보여준다.

### 완료 기준
- 현재 진행 상태가 문제 풀이 중 실시간으로 반영된다.

---

## 8.6 다음 문제 이동

### 목적
문제를 푼 뒤 자연스럽게 다음 문제로 넘어갈 수 있어야 한다.

### 요구사항
- 채점 전에는 다음 문제 버튼이 비활성화되거나 숨김 처리된다.
- 채점 후 다음 문제 버튼이 활성화된다.
- 마지막 문제가 아니면 다음 문제로 이동한다.
- 마지막 문제면 결과 화면으로 이동한다.

### 완료 기준
- 문제 이동 흐름이 끊기지 않는다.
- 마지막 문제 이후 결과 화면으로 정확히 이동한다.

---

## 8.7 결과 화면

### 목적
세션 결과를 이해하기 쉽게 보여주고, 재도전을 유도해야 한다.

### 요구사항
- 총 문제 수를 보여준다.
- 정답 수를 보여준다.
- 오답 수를 보여준다.
- 정답률을 보여준다.
- 틀린 문제 목록을 보여준다.
- 오답이 있으면 다시 풀기 버튼을 보여준다.
- 처음으로 버튼을 보여준다.
- 오답이 없으면 축하 문구를 보여준다.

### 완료 기준
- 세션 결과가 정확하게 표시된다.
- 오답 리스트가 올바르게 출력된다.

---

## 8.8 오답 다시 풀기

### 목적
틀린 문제를 바로 복습할 수 있어야 한다.

### 요구사항
- 이전 세션에서 틀린 문제만 다시 문제 배열로 만든다.
- 새 세션처럼 다시 시작한다.
- 다시 풀기 세션에서도 점수와 오답 기록이 새로 반영된다.

### 완료 기준
- 오답 문제만 재출제된다.
- 다시 풀기 흐름이 정상 동작한다.

2. 한 문제는 한 번만 채점한다.
3. 다음 문제로 넘어갈 때 현재 문제 상태를 초기화한다.
4. 오답 다시 풀기는 새 세션처럼 동작한다.

## 16. 예외 처리 요구사항

### 16.1 단 선택 없이 시작
- 시작 버튼 비활성화 또는 동작 금지

### 16.2 답 중복 선택
- 이미 채점된 문제는 추가 입력 무시

### 16.3 오답이 없을 때 다시 풀기
- 다시 풀기 버튼 숨김 또는 비활성화

### 16.4 마지막 문제 처리
- 인덱스 초과 없이 결과 화면으로 이동

## 17. 수동 테스트 시나리오

최소 아래 항목은 반드시 테스트한다.

1. 단을 선택하지 않으면 시작이 안 되는지
2. 단 선택 후 정상적으로 문제 화면으로 이동하는지
3. 객관식 선택 시 즉시 채점되는지
4. 같은 문제에서 두 번 점수 반영이 안 되는지
5. 10문제 종료 후 결과 화면이 나오는지
6. 오답 리스트가 정확히 보이는지
7. 오답 다시 풀기가 정상 동작하는지
8. 모바일 화면에서 버튼이 잘 눌리는지
9. 정답/오답 피드백 문구가 자연스러운지
