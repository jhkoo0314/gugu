# 초등학생 구구단 연습장 디자인 시스템
버전: v0.1  
기준 문서: PRD.md, TECH_SPEC.md, AGENTS.md  
대상: 디자이너, 프론트엔드 개발자, AI 코딩 에이전트  
목표: 초등학생 여자 아이가 좋아할 수 있는 밝고 예쁜 분위기를 유지하면서도, 문제 풀이에 집중할 수 있는 일관된 UI 시스템 정의

---

## 1. 문서 목적

이 문서는 구구단 연습장의 시각적 규칙과 UI 구성 원칙을 정의한다.

이 문서를 통해 아래를 일관되게 유지한다.

- 전체 브랜드 분위기
- 색상 사용 방식
- 타이포그래피 구조
- 여백과 레이아웃 규칙
- 컴포넌트 스타일 기준
- 상태별 시각 표현
- 반응형 UI 규칙
- 접근성과 사용성 기준

핵심 목표는 다음 한 줄로 정리할 수 있다.

**“딸기우유와 라벤더 색감의 밝고 폭신한 학습 놀이터”**

---

## 2. 브랜드 방향

## 2.1 제품 인상
사용자가 처음 봤을 때 느껴야 할 인상은 아래와 같다.

- 밝다
- 예쁘다
- 귀엽다
- 안전하다
- 어렵지 않다
- 눌러보고 싶다

## 2.2 디자인 핵심 가치
이 디자인 시스템은 아래 4가지를 우선한다.

1. 예쁘다  
2. 쉽다  
3. 집중된다  
4. 부담이 없다  

## 2.3 톤 앤 무드
권장 톤은 아래와 같다.

- 딸기우유 핑크
- 라벤더 퍼플
- 따뜻한 크림 배경
- 작은 민트 포인트
- 별과 반짝이 느낌의 장식
- 둥글고 말랑한 카드
- 도톰하고 누르기 쉬운 버튼

## 2.4 금지 방향
아래 방향은 피한다.

- 형광색 중심 디자인
- 진한 원색 남발
- 지나치게 아기 같은 공주풍
- 너무 많은 캐릭터와 장식
- 문제 풀이보다 배경이 더 눈에 띄는 구성
- 오답에 강한 빨간색 경고 UI 사용

---

## 3. 디자인 원칙

## 3.1 한 화면 한 행동
한 화면에는 핵심 행동 1개만 분명하게 보여야 한다.

- 시작 화면: 단 선택
- 문제 화면: 답 고르기
- 결과 화면: 결과 보기 / 다시 풀기

## 3.2 큰 글씨, 큰 버튼
초등학생 대상 서비스이므로 작은 텍스트나 작은 클릭 영역은 금지한다.

## 3.3 귀엽되 산만하지 않게
장식은 포인트로만 사용하고, 학습 흐름을 방해하면 안 된다.

## 3.4 오답도 부드럽게
오답 표시도 아이를 위축시키지 않는 톤으로 표현해야 한다.

## 3.5 카드 중심 구조
콘텐츠는 가급적 카드에 담아 안정적으로 보여준다.

## 4. 컬러 시스템

## 4.1 컬러 전략
색은 많아 보여도 실제 핵심 역할은 명확하게 나눈다.

- 배경 컬러
- 텍스트 컬러
- 메인 액션 컬러
- 보조 액션 컬러
- 성공 / 오답 / 강조 상태 컬러

---

### Background
- `bg.base = #FFF8F3`
- `bg.surface = #FFFFFF`
- `bg.soft = #FFF1F6`

### Brand
- `brand.primary = #FF8FB1`
- `brand.primaryHover = #FF7AA3`
- `brand.secondary = #C9B6FF`
- `brand.secondaryHover = #B9A0FF`

### Accent
- `accent.mint = #BEEDE0`
- `accent.yellow = #FFD96A`
- `accent.peach = #FFC9A9`

### Text
- `text.primary = #3F3650`
- `text.secondary = #5B5566`
- `text.muted = #8B8398`
- `text.onPrimary = #FFFFFF`

### State
- `state.success = #7ED6A7`
- `state.successSoft = #EAFBF1`
- `state.error = #FFB3C1`
- `state.errorSoft = #FFF0F4`
- `state.disabled = #D9D9E3`
- `state.border = #EADFE8`

---

## 4.3 추천 사용 비율
화면 내 권장 비중은 아래와 같다.

- 크림/화이트 배경: 60~70%
- 핑크/라벤더 포인트: 15~20%
- 민트/옐로 강조: 5~10%
- 텍스트/라인: 나머지

원칙:
- 핑크를 전부에 바르지 않는다.
- 라벤더, 크림, 화이트로 숨 쉴 공간을 만든다.

---

## 4.4 상태별 컬러 규칙

### 정답
- 배경: `state.successSoft`
- 포인트: `state.success`
- 텍스트: `text.primary`

### 오답
- 배경: `state.errorSoft`
- 포인트: `state.error`
- 텍스트: `text.primary`

### 선택 상태
- 배경: 연한 핑크 또는 연한 라벤더
- 보더: `brand.primary` 또는 `brand.secondary`

### 비활성화
- 배경: `state.disabled`
- 텍스트: `text.muted`

## 5. Tailwind 토큰 권장 맵

프로젝트에서는 커스텀 토큰을 `globals.css` 또는 Tailwind theme 확장으로 관리하는 것을 권장한다.

예시 의미 매핑:

- `--bg-base: #FFF8F3`
- `--bg-surface: #FFFFFF`
- `--brand-primary: #FF8FB1`
- `--brand-secondary: #C9B6FF`
- `--text-primary: #3F3650`
- `--text-secondary: #5B5566`
- `--state-success: #7ED6A7`
- `--state-error: #FFB3C1`

Tailwind utility 사용 기준:
- 배경: `bg-white`, `bg-rose-50`, `bg-violet-50` 유사 톤
- 텍스트: `text-slate-700`보다 조금 더 부드러운 보라/그레이 계열 커스텀 권장
- 보더: `border-rose-100`, `border-violet-100` 유사 톤 권장

## 6. 타이포그래피 시스템

## 6.1 원칙
텍스트는 귀엽게 보이되 읽기 쉬워야 한다.

우선순위:
1. 가독성
2. 숫자 인지성
3. 둥근 분위기

## 6.2 사용 원칙
- 너무 장식적인 손글씨체 금지
- 숫자가 헷갈리는 폰트 금지
- 본문은 편안하고 선명해야 함

## 6.3 텍스트 계층

### Display / 메인 타이틀
- 용도: 시작 화면 제목, 결과 화면 축하 제목
- 권장 크기: `text-3xl` ~ `text-4xl`
- 권장 두께: `font-extrabold` 또는 `font-bold`
- 컬러: `text.primary`

### H1 / 화면 제목
- 용도: “오늘의 결과”, “몇 단을 연습할까?”
- 권장 크기: `text-2xl`
- 권장 두께: `font-bold`

### H2 / 섹션 제목
- 용도: 오답 목록 제목, 점수 카드 제목
- 권장 크기: `text-xl`
- 권장 두께: `font-semibold`

### Body Large
- 용도: 설명 문구, 칭찬 문구
- 권장 크기: `text-lg`
- 권장 두께: `font-medium`

### Body
- 용도: 일반 안내 텍스트
- 권장 크기: `text-base`
- 권장 두께: `font-normal`

### Caption
- 용도: 보조 설명, 작은 상태 텍스트
- 권장 크기: `text-sm`
- 권장 두께: `font-medium`

### Problem Number
- 용도: 구구단 문제 식
- 권장 크기: `text-4xl` ~ `text-5xl`
- 권장 두께: `font-extrabold`

---

## 6.4 줄간격 기준
- 큰 제목: `leading-tight`
- 일반 본문: `leading-relaxed`
- 버튼 텍스트: `leading-none` 또는 `leading-tight`

---

## 7. 간격 시스템

## 7.1 기본 간격 규칙
화면이 답답하지 않도록 충분한 여백을 확보한다.

권장 간격 스케일:
- `4px`
- `8px`
- `12px`
- `16px`
- `20px`
- `24px`
- `32px`

## 7.2 섹션 간격
- 카드 내부 주요 블록 간: `16px ~ 20px`
- 화면 내 큰 구간 간: `24px ~ 32px`
- 버튼 사이 간격: `12px ~ 16px`

## 7.3 패딩 기준
- 메인 카드: `p-6` 기본
- 작은 카드: `p-4`
- 버튼: `px-5 py-3` 이상 권장

---

## 8. 라운드와 쉐도우 시스템

## 8.1 라운드 원칙
이 서비스는 둥근 인상을 유지해야 한다.

권장 라운드:
- 카드: `rounded-3xl`
- 버튼: `rounded-2xl` 또는 `rounded-full`
- 작은 배지/태그: `rounded-full`

## 8.2 쉐도우 원칙
그림자는 부드럽고 가벼워야 한다.

권장:
- 카드: `shadow-lg`
- 작은 카드/버튼: `shadow-md`
- hover 시 그림자 약간 증가 가능

금지:
- 너무 진하고 딱딱한 그림자
- 여러 겹의 과한 그림자

---

## 9. 보더 시스템

## 9.1 기본 규칙
보더는 강하게 쓰지 않고, 필요할 때만 부드럽게 사용한다.

권장:
- `1px` 보더
- 매우 옅은 핑크/보라/그레이 계열

## 9.2 사용 위치
- 선택된 버튼 강조
- 카드 구분
- 입력/선택 영역 분리

금지:
- 두꺼운 검정 보더
- 강한 대비의 딱딱한 외곽선

---

## 10. 레이아웃 시스템

## 10.1 전체 페이지 구조
기본 페이지 구조는 아래를 따른다.

- 전체 배경
- 중앙 정렬된 메인 컨테이너
- 카드형 콘텐츠 블록
- 하단 액션 버튼

## 10.2 페이지 래퍼
권장 스타일:
- `min-h-screen`
- `flex items-center justify-center`
- `px-4 py-6`
- 최대 폭 제한

## 10.3 메인 컨테이너
권장 스타일:
- `w-full max-w-md` 또는 `max-w-lg`
- 중앙 정렬
- 카드형 배치

## 10.4 반응형 규칙
모바일 우선 기준:
- 모바일: 기본 레이아웃
- 태블릿 이상: 넓어져도 콘텐츠 폭은 제한
- 데스크톱: 너무 넓게 퍼지지 않음

---

## 11. 배경 시스템

## 11.1 기본 배경
배경은 화사하되 시선 경쟁이 없어야 한다.

권장:
- 크림톤 단색 배경
- 아주 약한 핑크/라벤더 그라데이션
- 옅은 반짝이 점 포인트

예시 방향:
- 상단 옅은 핑크
- 하단 크림
- 매우 낮은 채도의 그라데이션

## 11.2 장식 배경
장식은 매우 약하게만 사용한다.

허용:
- 흐린 별
- 작은 하트
- 반짝이 점
- 구름 모양 blur

금지:
- 큰 패턴
- 반복 무늬
- 문제 카드와 경쟁하는 장식

---

## 12. 아이콘 시스템

## 12.1 방향
아이콘은 귀엽고 단순해야 한다.

추천 키워드:
- 별
- 하트
- 리본
- 반짝이
- 구름
- 스마일

## 12.2 사용 규칙
- 한 화면에서 아이콘 수 과다 금지
- 동일 스타일 세트 사용
- 텍스트보다 먼저 눈에 띄면 안 됨

## 12.3 사용 위치
- 제목 옆
- 결과 축하 영역
- 점수 카드
- 버튼 앞 작은 포인트

---

## 13. 모션 시스템

## 13.1 모션 원칙
모션은 “재미를 더하는 보조 요소”이며, 집중을 방해하면 안 된다.

## 13.2 허용 모션
- 버튼 hover 시 약한 scale-up
- 카드 등장 시 fade-up
- 정답 시 작은 반짝이 강조
- 결과 화면 진입 시 부드러운 등장

## 13.3 금지 모션
- 큰 bounce
- 지속 반복 애니메이션
- 화면 전체 흔들림
- 빠른 깜빡임
- 장시간 시선 고정 유도 효과

## 13.4 권장 전환 톤
- 짧고 부드럽게
- 150ms ~ 250ms 수준
- easing은 부드러운 curve 권장

---

## 14. 콘텐츠 톤 시스템

## 14.1 문구 원칙
모든 문구는 아래 기준을 따른다.

- 짧아야 한다
- 따뜻해야 한다
- 쉬워야 한다
- 혼내지 않아야 한다

## 14.2 정답 문구
예시:
- 정답이에요!
- 잘했어요!
- 멋져요!
- 맞았어요!

## 14.3 오답 문구
예시:
- 괜찮아요, 다시 해봐요!
- 조금만 더 생각해볼까?
- 아쉽지만 괜찮아요!
- 정답은 24예요!

## 14.4 결과 문구
예시:
- 오늘도 열심히 했어요!
- 끝까지 풀어서 정말 멋져요!
- 다시 도전하면 더 잘할 수 있어요!
- 모두 맞았어요! 정말 대단해요!

## 15. 컴포넌트 스타일 시스템

## 15.1 Page Shell
역할:
- 전체 화면의 공통 바탕

스타일 원칙:
- `min-h-screen`
- 따뜻한 배경
- 중앙 정렬
- 적당한 상하 여백

---

## 15.2 Main Card
역할:
- 시작/문제/결과 화면의 핵심 컨테이너

스타일 원칙:
- 화이트 또는 매우 옅은 크림 배경
- `rounded-3xl`
- `shadow-lg`
- `p-6`
- 부드러운 보더 가능

예상 느낌:
- 떠 있는 학습 카드
- 안전하고 정돈된 인상

---

## 15.3 Primary Button
용도:
- 시작하기
- 다음 문제
- 오답 다시 풀기

스타일 원칙:
- 메인 핑크 배경
- 흰색 텍스트
- 도톰한 높이
- 둥근 pill 또는 soft rounded
- hover 시 살짝 진해짐 또는 살짝 커짐

권장 속성:
- `font-semibold`
- `min-h-[52px]` 이상
- `rounded-full`
- 충분한 좌우 패딩

---

## 15.4 Secondary Button
용도:
- 처음으로
- 보조 동작

스타일 원칙:
- 라벤더 또는 화이트 배경
- 은은한 보더
- 메인 버튼보다 낮은 시각 우선순위

---

## 15.5 Dan Select Button
용도:
- 2단~9단, 전체 선택

기본 상태:
- 화이트/크림 배경
- 옅은 보더
- 어두운 텍스트

선택 상태:
- 핑크 또는 라벤더 tint 배경
- 강조 보더
- 살짝 그림자 증가

원칙:
- 선택 여부가 한눈에 보여야 함
- 너무 작은 버튼 금지

---

## 15.6 Choice Button
용도:
- 객관식 보기 버튼

기본 상태:
- 큰 터치 영역
- 세로 정렬
- 화이트 배경
- 부드러운 보더

정답 상태:
- 민트/연초록 배경
- 긍정적 강조

오답 상태:
- 연분홍 배경
- 부드러운 안내

비활성 상태:
- 클릭 방지
- 너무 흐리게 만들지 말고 읽힘 유지

---

## 15.7 Progress Bar
역할:
- 현재 진행률 표시

스타일 원칙:
- 트랙은 옅은 크림/그레이
- 진행 부분은 핑크 또는 라벤더
- 너무 얇지 않게
- 둥근 모서리 유지

---

## 15.8 Score Card / ScoreBoard
역할:
- 정답 수, 오답 수, 정답률 표시

스타일 원칙:
- 작은 카드 2~3개로 분리 가능
- 연한 배경색 사용
- 숫자는 크게
- 설명 텍스트는 짧게

추천:
- 정답: 민트 포인트
- 오답: 연핑크 포인트
- 정답률: 옐로 또는 라벤더 포인트

---

## 15.9 Feedback Message
역할:
- 정답/오답 즉시 피드백

스타일 원칙:
- 메시지 길이 짧게
- 배경 tint 사용 가능
- 정답/오답 상태 명확
- 경고창처럼 보이지 않게

---

## 15.10 Wrong Answer List
역할:
- 결과 화면의 오답 목록

스타일 원칙:
- 줄마다 부드러운 카드 또는 분리형 리스트
- 식, 정답, 내 답이 쉽게 읽혀야 함
- 정보 과다 금지

---

## 16. 화면별 스타일 가이드

## 16.1 시작 화면
목표 분위기:
- 설렘
- 밝음
- 시작하고 싶음

스타일 포인트:
- 제목이 가장 눈에 띄어야 함
- 단 선택 버튼은 카드형 그리드
- 시작 버튼은 가장 강한 액션 컬러

권장 요소:
- 작은 반짝이 장식
- 아주 짧은 설명 문구
- 넉넉한 여백

---

## 16.2 문제 화면
목표 분위기:
- 집중
- 가벼운 긴장감
- 즉시 피드백

스타일 포인트:
- 문제 식이 화면에서 가장 커야 함
- 보기 버튼은 세로 배열
- 진행률은 상단에 간단히
- 채점 결과가 부드럽게 나타남

금지:
- 화면 중간에 너무 많은 부가 정보
- 보기 버튼이 작거나 촘촘한 배열

---

## 16.3 결과 화면
목표 분위기:
- 뿌듯함
- 칭찬
- 다시 도전하고 싶음

스타일 포인트:
- 정답 수, 오답 수, 정답률을 크게
- 축하 메시지를 먼저 보여줌
- 오답은 부담 없이 정리
- 다시 풀기 버튼은 눈에 잘 띄게

---

## 17. 반응형 시스템

## 17.1 기준
모바일 우선을 기본으로 한다.

### 모바일
- 기본 레이아웃
- 버튼 세로 정렬 우선
- 양쪽 여백 확보

### 태블릿
- 동일 구조 유지
- 카드 폭만 소폭 확장

### 데스크톱
- 콘텐츠 최대 폭 제한
- 중앙 카드 구조 유지
- 과도하게 넓어지지 않음

## 17.2 터치 기준
- 버튼 최소 높이 충분히 확보
- 보기 버튼 간 간격 충분히 확보
- 터치 오작동 방지

---

## 18. 접근성 규칙

## 18.1 명도 대비
연한 색 위에서도 텍스트는 충분히 읽혀야 한다.

금지:
- 연핑크 위 연보라 텍스트
- 장식 때문에 텍스트 가독성 저하

## 18.2 상태 전달
정답/오답은 색만으로 구분하지 않는다.

같이 써야 하는 요소:
- 텍스트
- 보더/아이콘
- 배경 차이

## 18.3 텍스트 길이
아이 대상이므로 문장은 짧고 단순해야 한다.

## 18.4 클릭 영역
버튼은 충분히 크게 유지한다.
