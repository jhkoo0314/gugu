# gugu 디자인 리뉴얼 가이드
버전: v0.4 디자인 리뉴얼 초안  
대상: 디자이너, 프론트엔드 개발자, AI 코딩 에이전트  
기준: v0.1 ~ v0.4 기능 구현 완료 상태

---

## 1. 문서 목적

이 문서는 현재 gugu의 기능은 유지한 채, 전체 비주얼을 더 귀엽고 사랑스럽고 기억에 남는 방향으로 리뉴얼하기 위한 디자인 기준을 정의한다.

이번 리뉴얼의 핵심은 아래와 같다.

1. 더 귀엽게
2. 더 통일감 있게
3. 더 아이 친화적으로
4. 더 제품답게

즉, 단순히 색을 더 넣는 것이 아니라, **배경 / 카드 / 버튼 / 장식 / 마이크로 인터랙션 / 마스코트**를 포함한 전체 경험을 하나의 테마로 묶는 것이 목적이다.

---

## 2. 리뉴얼 목표

현재 gugu는 기능적으로 충분히 완성도가 높다.  
이제 리뉴얼 목표는 “잘 작동하는 앱”에서 “아이가 다시 켜고 싶은 앱”으로 넘어가는 것이다.

### 이번 리뉴얼의 목표
- 첫 화면 인상이 더 예뻐야 한다.
- 결과 화면이 더 뿌듯해야 한다.
- 대시보드/오답노트/배지 화면이 더 귀여워야 한다.
- 카드/버튼/상태 UI가 더 통일감 있어야 한다.
- 작은 장식과 반응으로 살아있는 느낌을 줘야 한다.

## 3. 전체 테마 정의

## 3.1 추천 테마
**딸기우유 별빛 스터디룸**

이 테마는 아래 두 가지를 결합한 방향이다.

- 딸기우유 같은 부드럽고 달콤한 핑크 톤
- 라벤더와 별빛 포인트가 있는 꿈같은 학습 공간

## 3.2 핵심 인상
사용자가 앱을 켰을 때 느껴야 하는 인상은 아래와 같다.

- 폭신하다
- 예쁘다
- 따뜻하다
- 반짝인다
- 어렵지 않다
- 공부가 아니라 놀이처럼 느껴진다

## 3.3 비주얼 키워드
- 딸기우유
- 라벤더
- 크림
- 별빛
- 하트
- 리본
- 말랑한 카드
- 작은 반짝이
- 귀여운 마스코트

## 4. 리뉴얼 핵심 원칙

## 4.1 색을 늘리기보다 역할을 분명히 한다
색이 많아 보이더라도 실제 역할은 고정해야 한다.

- 핑크 = 메인 액션
- 라벤더 = 보조 액션
- 민트 = 정답 / 성공
- 옐로 = 배지 / 성취
- 피치 = 경고 / 타이머 주의

## 4.2 귀여움은 장식보다 구조에서 나온다
아래 3개가 먼저 좋아야 한다.

1. 둥근 카드
2. 누르고 싶은 버튼
3. 부드러운 배경

장식은 그 다음이다.

## 4.3 공부 흐름을 해치지 않는다
아무리 귀여워도 아래를 해치면 안 된다.

- 문제 식 가독성
- 버튼 터치성
- 결과 정보 전달력
- 오답노트 읽기 편함

## 4.4 장식은 “작게, 정확하게”
하트, 별, 리본은 많을수록 귀여운 게 아니라  
**필요한 지점에만 작게 써야** 더 예쁘다.

## 5. 비주얼 시스템 리뉴얼 방향

## 5.1 배경 레이어 추가
현재가 단순 배경이라면, 리뉴얼 후에는 최소 2~3개 배경 레이어를 가진다.

### 레이어 1: 기본 배경
- 크림톤 단색 또는 매우 약한 그라데이션

### 레이어 2: 큰 블러 장식
- 좌상단 핑크 블러
- 우하단 라벤더 블러

### 레이어 3: 작은 반짝이 포인트
- 화면 모서리 근처 별 점
- 카드 바깥쪽의 작은 장식

### 주의
- 문제 카드 뒤에 직접 겹치지 않게
- 텍스트 뒤쪽 장식 금지
- 장식은 흐릿하고 약해야 함

---

## 5.2 카드 시스템 업그레이드
모든 주요 정보는 카드로 표현하되, 카드별 성격을 더 분명히 나눈다.

### 카드 공통 규칙
- 둥근 큰 모서리
- 화이트 또는 옅은 tint 배경
- 연한 핑크/보라 보더
- 말랑한 그림자
- 넉넉한 패딩

### 카드 유형

#### A. 기본 카드
- 일반 정보
- 오답 리스트
- 설정 블록

#### B. 강조 카드
- 결과 요약
- 오늘의 추천
- 새 배지 획득

#### C. 장식 카드
- 빈 상태 화면
- 축하 카드
- streak 보상 카드

---

## 5.3 버튼 시스템 업그레이드
버튼은 서비스의 귀여움을 가장 크게 좌우한다.

### 공통 규칙
- 키가 크고 도톰해야 함
- 너무 네모나지 않아야 함
- 눌렀을 때 약간 들어가는 느낌
- 색만 바뀌는 것이 아니라 그림자도 반응

### 메인 버튼
예:
- 시작하기
- 다시 풀기
- 추천 단 시작
- 오답노트 복습 시작

스타일 방향:
- 핑크 그라데이션 또는 핑크 단색
- 흰색 텍스트
- 둥근 pill 형태
- 약한 glow 또는 그림자

### 보조 버튼
예:
- 처음으로
- 전체 보기
- 설정 토글

스타일 방향:
- 라벤더/화이트
- 연한 보더
- 부드러운 텍스트

### 선택 버튼
예:
- 단 선택
- 답안 모드 선택
- 타이머 시간 선택
- 오답노트 필터

스타일 방향:
- 기본은 크림/화이트
- 선택되면 핑크 tint 또는 라벤더 tint
- 선택된 상태에 ring 추가 가능

## 7. 타이포그래피 리뉴얼

## 7.1 목표
글씨도 귀엽지만 읽기 쉬워야 한다.  
특히 숫자가 중요한 서비스이므로 숫자 인지성이 가장 중요하다.

## 7.2 스타일 방향
- 제목은 둥글고 힘 있게
- 문제 숫자는 크고 또렷하게
- 본문은 부드럽고 안정적이게
- 버튼 텍스트는 짧고 선명하게

## 7.3 우선순위
1. 문제 식
2. 결과 숫자
3. 버튼 텍스트
4. 카드 제목
5. 설명 문구

## 7.4 강조 방식
리뉴얼 후에는 폰트 자체보다 아래가 더 중요하다.

- 크기 차이
- 두께 차이
- 색 차이
- pill/배지 배경 차이

## 8. 마스코트 시스템

## 8.1 도입 방향
앱 전체에 작은 마스코트 1종을 반복적으로 등장시킨다.

### 추천 후보
- 토끼
- 별요정
- 곰젤리
- 리본 고양이

가장 추천:
**작은 토끼 또는 별요정**

이유:
- 초등학생 여자 아이 취향에 잘 맞음
- 말풍선 연출이 자연스러움
- 오답노트/배지/결과 화면과 연결하기 쉬움

## 8.2 사용 원칙
- 화면마다 크게 등장시키지 않는다.
- 작은 보조 캐릭터로만 사용한다.
- 앱의 주인공은 여전히 문제와 버튼이다.

## 8.3 사용 위치
- 시작 화면 타이틀 옆
- 결과 화면 축하 헤더
- 오답노트 empty state
- 새 배지 획득 배너
- 오늘의 추천 카드

## 8.4 말풍선 문구 예시
- 오늘도 같이 연습해볼까?
- 정말 잘했어!
- 이 문제는 다시 해보자!
- 새 배지를 얻었어!

---

## 9. 화면별 리뉴얼 전략

## 9.1 시작 화면

### 목표
앱에 들어오자마자 “예쁘다, 눌러보고 싶다”는 느낌을 줘야 한다.

### 리뉴얼 포인트
- 상단에 큰 타이틀 + 작은 반짝이
- 마스코트 1종 추가
- 단 선택 버튼을 캔디 카드처럼 변경
- 시작 버튼을 가장 도톰하고 예쁘게 강조
- 배경에 약한 별빛 장식

### 추가 가능 요소
- “오늘은 몇 단을 연습해볼까?” 문구
- 최근 추천 단 카드
- 최근 배지 요약 1개

---

## 9.2 퀴즈 화면

### 목표
귀엽지만 집중이 깨지지 않게 해야 한다.

### 리뉴얼 포인트
- 문제 카드를 크게, 더 깨끗하게
- 진행률 바를 젤리형 또는 말랑한 pill형으로
- 타이머를 딱딱한 숫자 대신 둥근 capsule UI로
- streak 보상 시 상단 리본형 배너 등장
- 피드백 메시지를 작은 칭찬 배지처럼 표시

### 주의
- 퀴즈 화면은 가장 덜 장식해야 한다.
- 장식보다 문제 식 가독성이 먼저다.

---

## 9.3 결과 화면

### 목표
가장 예쁘고 가장 뿌듯해야 한다.

### 리뉴얼 포인트
- 큰 축하 헤더
- 반짝이/별 장식
- 결과 숫자를 카드형으로 크게 표시
- 새 배지 획득 카드
- 최고 streak 강조
- 추천 액션 버튼을 예쁘게 배치

### 추천 구조
1. 축하 헤더
2. 결과 요약 카드
3. 새 배지/성과 카드
4. 오답 요약
5. 다시 풀기 / 홈 버튼

결과 화면은 **디자인 투자 우선순위 1순위**다.

---

## 9.4 대시보드 화면

### 목표
학습 서비스처럼 보여야 한다.

### 리뉴얼 포인트
- “오늘의 추천” 카드 상단 배치
- 단별 성취도 카드를 스티커북처럼 정리
- 약한 단 카드에 귀여운 강조 뱃지
- 최근 배지 요약 카드
- 오답노트 남은 문제 수 요약 카드

### 상태 카드 문구 예시
- 다시 연습해볼까?
- 거의 다 왔어요!
- 정말 잘하고 있어요!
- 아직 시작 전이에요!

---

## 9.5 오답노트 화면

### 목표
오답을 무겁지 않게, 다시 풀고 싶게 보여야 한다.

### 리뉴얼 포인트
- 경고 느낌 대신 복습 노트 느낌
- 연핑크/라벤더 기반 리스트
- 상태 뱃지를 귀엽게 표현
- 해결 완료 항목은 부드럽게 흐리게 표시

### empty state
아주 중요하다.

예시:
- “오답노트가 비어 있어요!”
- “정말 잘하고 있어요!”
- 작은 토끼/별요정 캐릭터 표시

---

## 9.6 배지 화면

### 목표
수집하는 재미를 느끼게 한다.

### 리뉴얼 포인트
- 배지 그리드를 스티커북 느낌으로
- 해금된 배지는 컬러풀하게
- 잠금 배지는 흐리고 부드럽게
- 획득일을 작은 태그로 표시

### 비주얼 키워드
- 리본
- 별
- 왕관
- 스티커
- 트로피
- 젤리 메달

---

## 10. empty state 시스템

## 10.1 목적
데이터가 없을 때도 화면이 귀엽고 친절하게 보여야 한다.

## 10.2 대상 화면
- 오답노트 비어있음
- 기록 없음
- 배지 없음
- 추천 단 없음
- 단별 성취도 데이터 없음

## 10.3 구성
- 작은 아이콘 또는 마스코트
- 짧은 제목
- 따뜻한 설명
- 다음 행동 버튼

## 10.4 예시

### 오답노트
- 제목: 오답노트가 비어 있어요!
- 설명: 정말 잘하고 있어요.
- 버튼: 새로운 퀴즈 시작하기

### 배지
- 제목: 첫 배지를 모아볼까?
- 설명: 문제를 풀면 배지를 받을 수 있어요.
- 버튼: 퀴즈 시작하기

### 학습 기록
- 제목: 첫 기록을 만들어볼까?
- 설명: 오늘의 연습을 시작하면 기록이 쌓여요.

## 11. 마이크로 인터랙션 가이드

## 11.1 목표
서비스를 더 살아있고 귀엽게 느끼게 한다.

## 11.2 허용 인터랙션
- 버튼 hover 시 약한 scale-up
- 버튼 tap 시 약한 scale-down
- 카드 등장 fade-up
- 정답 시 작은 sparkle
- streak 달성 시 리본 배너 등장
- 배지 획득 시 작은 반짝이 팝업

## 11.3 금지 인터랙션
- 화면 전체 bounce
- 과도한 confetti
- 계속 움직이는 배경
- 너무 빠른 깜빡임
- 긴 애니메이션

## 11.4 권장 속도
- 150ms ~ 250ms
- 부드러운 easing 사용

## 16. 리뉴얼 후 기대 효과

이번 디자인 리뉴얼이 잘 적용되면 아래 변화가 생긴다.

1. 첫 화면 인상이 훨씬 더 강해진다.
2. 결과 화면 만족도가 올라간다.
3. 대시보드/오답노트/배지 기능이 더 살아난다.
4. 단순 기능 앱이 아니라 작은 학습 브랜드처럼 보이게 된다.
5. 재방문하고 싶은 느낌이 생긴다.
