# FEATURE_SPEC_V3.md

# gugu 학습 확장 기능 명세서
버전: v0.4 초안  
대상: 기획자, 프론트엔드 개발자, AI 코딩 에이전트  
기준: v0.1 ~ v0.3 완료 상태

---

## 1. 문서 목적

이 문서는 현재 gugu에 다음 학습 확장 기능을 추가하기 위한 요구사항과 설계 기준을 정의한다.

대상 기능:
- 단별 성취도
- 약점 단 추천
- 오답노트
- 배지 시스템

이 기능들은 단순히 화면을 늘리는 목적이 아니라, 아래 학습 흐름을 완성하는 데 목적이 있다.

1. 사용자가 문제를 푼다.
2. 기록이 쌓인다.
3. 단별 성취도를 계산한다.
4. 약한 단을 자동 추천한다.
5. 틀린 문제를 누적 관리한다.
6. 반복 학습과 성취 보상을 제공한다.

즉, 이 문서는 gugu를 단순 퀴즈 앱에서 **지속 사용 가능한 학습형 서비스**로 확장하기 위한 기준 문서다.

---

## 2. 확장 목표

이번 버전의 목표는 아래와 같다.

1. 사용자가 어떤 단을 잘하고 못하는지 한눈에 볼 수 있어야 한다.
2. 앱이 “오늘은 무엇을 다시 해야 하는지” 추천할 수 있어야 한다.
3. 틀린 문제를 결과 화면이 아닌 별도 학습 자산으로 누적 관리할 수 있어야 한다.
4. 학습을 계속하고 싶게 만드는 보상 장치를 제공해야 한다.

---

## 3. 핵심 개념

이번 버전의 핵심 개념은 아래 4개다.

### 3.1 성취도
단별로 얼마나 연습했고, 얼마나 정확하게 맞혔는지를 수치로 보여주는 기능

### 3.2 약점 추천
최근 기록과 성취도를 바탕으로 다시 연습해야 할 단을 추천하는 기능

### 3.3 오답노트
틀린 문제를 세션 단위가 아니라 누적 학습 데이터로 저장하고 다시 복습하는 기능

### 3.4 배지
학습 횟수, 정답률, 연속 학습, 특정 성취 조건을 달성했을 때 보상하는 기능

# 5. 단별 성취도

## 5.1 목적
2단부터 9단까지 각 단의 학습 상태를 정리해서 보여준다.

## 5.2 사용자 가치
- 내가 어떤 단을 잘하는지 알 수 있다.
- 어떤 단이 약한지 바로 이해할 수 있다.
- 막연한 반복이 아니라 목표 있는 연습이 가능해진다.

## 5.3 표시 대상
2단 ~ 9단

## 5.4 단별 성취도 정의
단별 성취도는 아래 데이터를 기반으로 계산한다.

- 총 풀이 수
- 정답 수
- 오답 수
- 정답률
- 최근 풀이 수
- 최근 정답률
- 최고 연속 정답
- 시간 초과 수(타이머 모드 사용 시)

## 5.5 권장 계산 기준

### 누적 기준
전체 학습 기록에서 해당 단 관련 문제를 모두 합산

### 최근 기준
최근 N세션 또는 최근 N문제 기준으로 별도 계산 가능

초기 구현은 아래를 권장한다.

- 누적 성취도 표시
- 최근 성취도는 선택 사항

## 5.6 데이터 구조 예시

```ts
type DanStats = {
  dan: number;
  totalSolved: number;
  correctCount: number;
  wrongCount: number;
  accuracy: number;
  timeoutCount: number;
  bestStreak: number;
  lastPlayedAt: string | null;
};
````

## 5.7 계산 방식

학습 기록과 문제 결과 데이터를 기반으로 단별로 집계한다.

정답률:

```ts
accuracy = totalSolved === 0 ? 0 : Math.round((correctCount / totalSolved) * 100)
```

## 5.8 UI 요구사항

단별 성취도는 카드형으로 보여주는 것을 권장한다.

각 카드에 표시:

* 단 이름 (예: 7단)
* 정답률
* 총 풀이 수
* 오답 수
* 최근 학습일 또는 최근 풀이 여부
* 상태 라벨

상태 라벨 예시:

* 시작 전
* 연습 중
* 잘하고 있어요
* 다시 연습 필요
* 거의 마스터

## 5.9 상태 분류 정책 예시

초기 기준 예시:

* `시작 전`: totalSolved = 0
* `다시 연습 필요`: accuracy < 60
* `연습 중`: accuracy 60~79
* `잘하고 있어요`: accuracy 80~94
* `거의 마스터`: accuracy >= 95 and totalSolved >= 10

## 5.10 화면 위치

권장 위치:

* 별도 `/dashboard` 페이지
* 또는 홈(`/`) 하단 요약 영역

초기 구현 권장:

* 홈 하단에 요약 카드 3개 정도
* 별도 `/dashboard` 페이지에서 전체 카드 상세 보기

# 6. 약점 단 추천

## 6.1 목적

사용자가 직접 고민하지 않아도 오늘 다시 연습하면 좋은 단을 추천한다.

## 6.2 사용자 가치

* “무엇을 해야 할지” 바로 알 수 있다.
* 반복 학습의 방향이 생긴다.
* 기록을 활용한 서비스처럼 느껴진다.

## 6.3 추천 대상

2단 ~ 9단 중 1~3개 추천

## 6.4 추천 기준

추천은 아래 우선순위를 기반으로 계산한다.

### 1순위

최근 정답률이 낮은 단

### 2순위

오답노트에 누적 문제 수가 많은 단

### 3순위

최근 시간 초과가 많았던 단

### 4순위

최근 학습하지 않은 단

## 6.5 추천 점수 예시

각 단마다 score를 계산해 가장 높은 순으로 추천한다.

예시:

```ts
weaknessScore =
  (100 - accuracy) * 0.5 +
  wrongNoteCount * 2 +
  timeoutCount * 2 +
  inactivityWeight
```

초기 구현은 더 단순하게 가도 된다.

권장 단순 기준:

1. accuracy 낮은 순
2. wrongNoteCount 많은 순
3. 최근 학습일 오래된 순

## 6.6 추천 결과 구조 예시

```ts
type RecommendedDan = {
  dan: number;
  reason: 'low-accuracy' | 'many-wrongs' | 'many-timeouts' | 'not-practiced-recently';
  label: string;
};
```

## 6.7 추천 문구 예시

* 최근 7단 정답률이 낮아요. 다시 연습해볼까요?
* 8단 오답이 많이 쌓였어요.
* 6단을 한동안 연습하지 않았어요.
* 9단에서 시간 초과가 자주 있었어요.

## 6.8 UI 요구사항

홈 화면 또는 대시보드 상단에 추천 영역을 둔다.

구성 예:

* 제목: 오늘의 추천
* 추천 카드 1~3개
* 추천 이유 짧은 문구
* 바로 시작 버튼

예:

* `7단 다시 풀기`
* `오답노트에서 8단 복습`

## 6.9 버튼 동작

추천 카드에서 바로 아래 액션 가능해야 한다.

* 해당 단으로 새 퀴즈 시작
* 오답노트 기반 복습 시작

## 6.10 구현 포인트

* 추천은 실시간 계산 또는 `useMemo`로 계산 가능
* 로컬 데이터 기반이므로 서버 없이 구현 가능

# 7. 오답노트

## 7.1 목적

세션이 끝나면 사라지는 오답이 아니라, 누적 학습 자산으로 오답을 관리한다.

## 7.2 사용자 가치

* 자주 틀리는 문제를 따로 모아 반복 복습할 수 있다.
* 단순 세션 결과가 아니라 실제 학습 기록처럼 느껴진다.
* 약점 단 추천과 직접 연결할 수 있다.

## 7.3 기능 요구사항

오답 문제를 localStorage에 누적 저장한다.

오답노트에서 할 수 있는 것:

* 전체 오답 목록 보기
* 단별 필터
* 특정 문제 다시 풀기
* 전체 다시 풀기
* 맞혀서 해결된 문제 표시 또는 제거

## 7.4 저장 단위

권장 구조:

```ts
type WrongNoteItem = {
  id: string;
  dan: number;
  multiplicand: number;
  multiplier: number;
  correctAnswer: number;
  lastUserAnswer: number | null;
  reason: 'wrong-answer' | 'timeout';
  wrongCount: number;
  resolvedCount: number;
  isMastered: boolean;
  lastWrongAt: string;
  lastResolvedAt: string | null;
};
```

## 7.5 저장 규칙

### 오답 발생 시

* 기존 동일 문제 있으면 `wrongCount + 1`
* 없으면 새 항목 생성
* `isMastered = false`

### 오답 문제를 다시 맞혔을 때

* `resolvedCount + 1`
* 일정 기준 충족 시 `isMastered = true`

## 7.6 마스터 처리 기준

초기 기준 권장:

* 같은 문제를 2회 이상 맞히면 `isMastered = true`
  또는
* 오답노트 복습에서 1회 맞히면 `isMastered = true`

초기 구현은 아래를 권장:

* 오답노트에서 다시 풀어 맞히면 `isMastered = true`

## 7.7 오답노트 페이지

권장 경로:

* `/wrong-note`

## 7.8 오답노트 화면 구성

영역 예시:

### 상단

* 제목
* 총 오답 문제 수
* 아직 해결 안 된 문제 수

### 필터

* 전체
* 2단 ~ 9단
* 해결 전만 보기
* 해결 완료 보기

### 리스트

* 문제 식
* 마지막 오답
* 오답 횟수
* 상태
* 다시 풀기 버튼

### 액션

* 전체 복습
* 필터 기준 복습
* 해결 완료 항목 숨기기

## 7.9 복습 방식

오답노트에서 복습 시작 시:

* 선택된 문제들만 새로운 문제 세트로 구성
* 일반 퀴즈 화면 재사용 가능

## 7.10 UI 상태 라벨 예시

* 새로 틀림
* 자주 틀림
* 복습 필요
* 해결 완료

## 7.11 구현 포인트

* 문제 식을 기준으로 고유 식별 가능
* 예: `3x7`
* 기존 `buildRetryQuestions` 로직을 확장해 재사용 가능

# 8. 배지 시스템

## 8.1 목적

학습을 계속하고 싶게 만드는 작은 보상 체계를 제공한다.

## 8.2 사용자 가치

* 성취감
* 반복 동기
* “내가 해냈다”는 느낌
* 학습 기록에 대한 보람

## 8.3 배지 설계 원칙

* 너무 많지 않게 시작한다.
* 조건이 명확해야 한다.
* 이름이 귀엽고 직관적이어야 한다.
* 학습을 압박하지 않고 칭찬하는 방향이어야 한다.

## 8.4 배지 유형

초기 배지는 아래 4개 그룹으로 나눈다.

### A. 시작 배지

* 첫 문제 풀이
* 첫 세션 완료
* 첫 오답노트 복습 완료

### B. 단별 배지

* 2단 연습 시작
* 특정 단 마스터
* 전체 단 모두 1회 이상 완료

### C. 성취 배지

* 정답률 100%
* 연속 정답 5회
* 시간 초과 없이 완료

### D. 꾸준함 배지

* 3일 연속 학습
* 5세션 달성
* 50문제 풀이

## 8.5 배지 타입 예시

```ts
type BadgeId =
  | 'first-start'
  | 'first-session-complete'
  | 'first-wrong-note-clear'
  | 'dan-2-start'
  | 'dan-master'
  | 'all-dan-tried'
  | 'perfect-score'
  | 'five-streak'
  | 'no-timeout-clear'
  | 'three-day-streak'
  | 'five-sessions'
  | 'fifty-solved';
```

## 8.6 배지 구조 예시

```ts
type Badge = {
  id: BadgeId;
  title: string;
  description: string;
  unlockedAt: string | null;
  isUnlocked: boolean;
};
```

## 8.7 배지 조건 예시

### first-start

첫 퀴즈 시작 시 해금

### first-session-complete

첫 세션 결과 화면 진입 시 해금

### first-wrong-note-clear

오답노트에서 문제를 처음 해결했을 때 해금

### dan-master

특정 단의 성취도가 일정 기준 이상일 때 해금
권장 기준:

* totalSolved >= 10
* accuracy >= 90

### all-dan-tried

2단~9단 모두 1회 이상 풀이

### perfect-score

10문제 세션에서 정답률 100%

### five-streak

한 세션 내 5연속 정답 달성

### no-timeout-clear

타이머 모드에서 시간 초과 없이 세션 완료

### three-day-streak

날짜 기준 3일 연속 세션 수행

### fifty-solved

누적 풀이 수 50문제 이상

## 8.8 UI 요구사항

### 위치

* 결과 화면에서 새로 획득한 배지 팝업/카드 표시
* 별도 `/badges` 페이지에서 전체 배지 목록 확인 가능
* 홈 또는 대시보드에서 최근 획득 배지 요약 가능

### 표시 방식

* 잠금/해금 상태 구분
* 이름
* 짧은 설명
* 획득일

## 8.9 보상 경험

새 배지를 얻었을 때:

* 결과 화면 상단 또는 중간에 강조
* 작은 별/반짝이 연출
* 효과음 연계 가능

## 8.10 구현 포인트

* 배지 해금은 결과 화면 진입 시점에 검사하는 것이 가장 단순하다.
* 일부 배지는 세션 시작/문제 해결 시점에 즉시 검사 가능하다.
* 중복 해금 방지 필요

# 10. 페이지/컴포넌트 확장 제안

* `/dashboard` : 단별 성취도 + 추천
* `/wrong-note` : 오답노트

## 10.3 새 컴포넌트 후보

# 13. 수동 테스트 항목

## 단별 성취도

* 각 단의 풀이 수가 정확히 집계되는지
* 정답률이 올바르게 계산되는지
* 풀이가 없는 단은 시작 전 상태로 보이는지

## 약점 단 추천

* 정답률이 낮은 단이 추천되는지
* 오답노트가 많은 단이 추천 우선순위에 반영되는지
* 추천 버튼으로 바로 학습 시작되는지

## 오답노트

* 틀린 문제가 누적 저장되는지
* 같은 문제를 여러 번 틀리면 wrongCount가 증가하는지
* 해결 완료 처리 로직이 정상 동작하는지
* 단별 필터가 올바르게 작동하는지

## 배지 시스템

* 첫 세션 완료 시 첫 배지가 열리는지
* 100점 달성 시 퍼펙트 배지가 열리는지
* 5연속 정답 시 streak 배지가 열리는지
* 이미 연 배지가 중복 지급되지 않는지

# 14. 완료 기준

다음 조건을 만족하면 v0.4 확장 구현이 완료된 것으로 본다.
